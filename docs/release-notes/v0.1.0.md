# DriftKB v0.1.0

First public MVP release of DriftKB, a local Python CLI for catching stale
Markdown knowledge-base docs when code changes.

## Highlights

- `driftkb validate` for curated KB freshness and verify-block checks.
- Constrained `rg` verify blocks with explicit expected assertions.
- Git diff checks that include committed, staged, unstaged, and untracked source
  changes.
- Fingerprint snapshots with explicit `driftkb fingerprints update`.
- Optional static call graph cache warning propagation.
- Advisory `driftkb gaps detect` and human-reviewed `driftkb promote`.
- Text and JSON reports for local workflows, hooks, and CI.
- Pre-push hook installation that calls `driftkb validate`.
- Optional `enterprise-java` profile and adapter.

## Install

```text
pipx install driftkb
uv tool install driftkb
```

## Safety

Verify blocks execute repository-local commands and should be reviewed like
scripts or tests. The MVP only executes constrained `rg` commands; use
`driftkb validate --no-verify` when reviewing untrusted changes.
